# 헤쳐 모여! (Fall In!)

## 개요

!!! tip "아이템 한줄 설명"
    6 Nimmt! 룰을 공군 ORI 세계관으로 재해석한 1인 전략 카드 게임

보드게임 **6 Nimmt!** 의 핵심 룰을 공군 ORI(작전준비태세) 세계관으로 재해석한
싱글 플레이 전략 카드 게임이다. Python + Pygame-CE로 1인 개발 중이며, 데스크톱
타깃(Windows / macOS / Linux) 배포를 목표로 한다.

### 저장소

<https://github.com/bnbong/Fall-In>

## 기획 의도

- 6 Nimmt! 특유의 "작은 수를 내도 크게 잃을 수 있는" 긴장감을 디지털에서 살리고 싶었다.
- 복무 중 자주 접하던 **ORI / 편성 / 집결 / 헤쳐모여** 같은 군 용어를 카드·페널티·라운드 구조에
  자연스럽게 녹이면 한국 플레이어에게 직관적인 세계관이 될 것이라 판단했다.
- 네트워크 없이도 혼자 플레이 가능하도록 **AI 봇 대전** 형태로 시작해, 추후 로컬 멀티로 확장할
  여지를 남겨두었다.

## 주요 역할

- 게임 룰 디지털화 (카드 덱 / 편성 / 페널티 계산 로직)
- Pygame-CE 기반 렌더링·입력 파이프라인 구성
- 씬/상태 머신 설계 (메인 → 전투 → 결과 → 복귀)
- AI 봇 전략 구현 (휴리스틱 기반, 난이도 단계화)
- 테스트(pytest) + GitHub Actions CI 파이프라인
- 아트/사운드 에셋 통합 및 라이선스 관리
- 멀티플랫폼 빌드 배포 자동화

## 핵심 설계 판단

### 1. 룰의 디지털화

아날로그 보드게임 룰을 그대로 옮기는 것이 아니라, **턴 단위 연출**과 **자동 정산**을 추가해
디지털에서만 가능한 몰입감을 확보했다. 대신 원작의 밸런스를 해치지 않도록 핵심 페널티 계산은
원작 규칙과 1:1로 맞췄다.

### 2. 책임 분리 구조

게임 상태(모델) / 규칙 엔진(서비스) / 렌더러(뷰)를 분리해 테스트를 작성하기 쉽게 했다.
규칙 엔진은 Pygame 의존성이 없어 pytest만으로 검증이 가능하다.

### 3. 멀티플랫폼 빌드

PyInstaller + Pygame-CE 조합으로 Windows/macOS/Linux 단일 실행 파일 빌드를
GitHub Actions 매트릭스로 구성했다. 릴리스 태그 푸시 → 3개 플랫폼 바이너리 자동 업로드.

## Stack

- Python 3.12
- Pygame-CE
- pytest (규칙 엔진 단위 테스트)
- GitHub Actions (CI + 멀티플랫폼 릴리스)
- PyInstaller (배포 바이너리)

## 결과 및 성과

- 규칙 엔진 pytest 커버리지 확보 (핵심 페널티/편성 로직)
- Windows / macOS / Linux 자동 빌드 릴리스 파이프라인 동작 확인
- 1인 개발 스코프 내에서 기획 → 구현 → 배포까지의 사이클 단축

## 배운 점

- **"게임은 프로덕트다"** — 단순 구현보다 릴리스 가능한 상태로 유지하는 것이 훨씬 큰 일이라는
  점을 체감했다. 특히 멀티플랫폼 빌드 CI 구성이 초기에 많은 시간을 소모했다.
- **도메인 언어의 힘** — 카드 이름/상태 명칭을 실제 공군 용어로 맞추니 코드 읽는 속도가 빨라졌고,
  기획-구현 간 의사소통 비용이 줄었다.
- **휴리스틱 AI의 재미 한계** — 단순 규칙 기반 AI의 예측 가능성이 금방 지루함으로 이어진다는 걸
  체감했다. 다음 개선은 MCTS 류의 탐색 기반으로 고려 중이다.
